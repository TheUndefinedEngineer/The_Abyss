## Introduction

It is about the history of C language and its standardisation.
- Its more of general language created with association to UNIX system
- Many of its ideas stem from the language BCPL which inspired B and then C, both of them are "typeless" languages.
- Greatly used for building compilers and operating systems which gave it the name "_system programming language_" even though its not tied to any specific OS or machine.
- Features of C:
	-  Control flow
	-  Switch cases
	-  Loops
	-  Break
	-  Data types
	-  Pointers
- Its relatively a "_low level language_" 
- Some cons:
	- Can't directly deal with "arrays", "strings", "sets" or "lists".
	- Doesn't define any storage allocation facility only static definition / stack discipline
	- No in-built **READ** or **WRITE** statements.
	- No in-built files access methods.
	- No **heap** or **garbage** collection.
- It offers only straightforward, single-thread control flow.
- Because of the absence of these features keeps the language down to a modest size which has its benefits.
